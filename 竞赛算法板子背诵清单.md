# 竞赛算法板子背诵清单

适用范围：ICPC / CCPC / 蓝桥杯 / 团体程序设计天梯赛。默认主语言为 C++。

更新时间：2026-04-27。

本文目标不是把所有代码模板塞进来，而是明确：

- 哪些板子必须背到能手写。
- 哪些板子要会套、会改边界。
- 哪些板子适合放进 ICPC/CCPC 队伍参考文档。
- 比赛中看到什么题面信号时优先想到它。

## 2. 比赛侧重点

| 比赛 | 主要难度来源 | 背板子重点 |
|---|---|---|
| 蓝桥杯 | 个人赛、实现、枚举、模拟、基础 DP、数学细节、日期/字符串/大数思维 | STL、排序、二分、前缀和、差分、DFS/BFS、并查集、最短路、背包、基础数论。高级图论和复杂字符串通常不是核心。 |
| 天梯赛 GPLT | 个人/团体总分，题目跨度大，L1/L2 偏实现，L3 可到图论/数据结构/DP | 输入输出、结构体排序、map/set、BFS/DFS、树、最短路、并查集、拓扑、基础 DP。目标高分再补 Tarjan、线段树、网络流。 |
| ICPC / CCPC | 三人一机，题型混合，思维题、构造、图论、字符串、数学、数据结构都可能出现 | 队伍模板要覆盖图论、数据结构、字符串、数学、计算几何、网络流；个人要能快速手写基础板子并知道高级板子怎么改。 |

建议分三级准备：

- A 级：必须背。比赛中不该翻资料，应该 3-8 分钟写出。
- B 级：必须会套。能从自己的模板本复制后改参数、改下标、改运算。
- C 级：会认题型。出现时知道查哪个模板、复杂度是什么、容易错哪里。

## 3. A 级：必须背到能手写

### 3.1 基础程序框架

| 板子 | 常见用法 | 注意点 |
|---|---|---|
| 快速 IO 框架 | 多组输入、读到 EOF、关闭同步 | `ios::sync_with_stdio(false); cin.tie(nullptr);` 后不要混用 `scanf/printf`。 |
| 类型别名与常量 | `ll`、`pii`、`INF`、`LINF`、`MOD` | 最短路距离用 `long long`；`INF + w` 可能溢出。 |
| 多组测试清空 | `vector.clear()`、数组初始化、时间戳数组 | 蓝桥杯/天梯赛常有多组或多个样例；别残留上组状态。 |
| 坐标离散化 | 值域大但只关心相对大小、区间端点压缩 | `sort + unique + lower_bound`；区间问题要考虑 `r+1`。 |
| 方向数组 | 网格 BFS/DFS、迷宫、连通块 | 四方向、八方向分清；边界判断统一写函数。 |
| 位运算基础 | 状压 DP、枚举子集、lowbit | 枚举子集：`for (int s=m; s; s=(s-1)&m)`。 |

### 3.2 STL 与常用技巧

| 板子/技巧 | 常见用法 | 注意点 |
|---|---|---|
| `sort` 自定义比较 | 结构体排序、贪心排序 | 比较函数必须满足严格弱序，不要写 `<=`。 |
| `lower_bound / upper_bound` | 二分位置、计数、离散化 | 只对有序区间使用；返回迭代器要转下标。 |
| `priority_queue` | Dijkstra、贪心取最大/最小 | 小根堆写 `greater<>`；Dijkstra 要跳过过期状态。 |
| `set / map` | 动态有序集合、去重、频次统计 | `erase(value)` 会删所有相同键；迭代器删除要小心。 |
| `unordered_map` | 大量哈希统计 | ICPC 中可能被卡哈希，必要时自定义 hash。 |
| `next_permutation` | 小规模全排列枚举 | 先排序；复杂度是阶乘级。 |

### 3.3 枚举、搜索、二分

| 板子 | 常见用法 | 注意点 |
|---|---|---|
| 二分答案 | 最小可行值、最大可行值、单调判定 | 关键是判定函数单调；整数二分注意死循环。 |
| 浮点二分 | 几何、平均值、概率阈值 | 迭代固定 60-100 次，别依赖 `l < r`。 |
| DFS 回溯 | 排列组合、分组、填数、剪枝搜索 | 全局答案回滚；剪枝顺序影响巨大。 |
| BFS 最短步数 | 网格、无权图、状态图 | 入队时标记 visited，避免重复入队爆炸。 |
| 双端 BFS | 边权只有 0/1 的最短路 | 用 `deque`；0 边 push_front，1 边 push_back。 |
| 折半搜索 | `n <= 40` 的子集和、方案计数 | 两半枚举后排序/双指针/二分合并。 |

### 3.4 前缀、差分、扫描线雏形

| 板子 | 常见用法 | 注意点 |
|---|---|---|
| 一维前缀和 | 区间和、子数组统计 | `pre[i] = pre[i-1] + a[i]`；区间 `[l,r]` 为 `pre[r]-pre[l-1]`。 |
| 二维前缀和 | 矩阵区域求和、子矩形枚举 | 容斥四项；下标从 1 开始最省事。 |
| 一维差分 | 区间加、最后还原数组 | `[l,r]` 加 `v`：`d[l]+=v,d[r+1]-=v`。 |
| 二维差分 | 矩形加、网格覆盖 | 四角更新，最后做二维前缀还原。 |
| 扫描线事件 | 区间覆盖、会议室、最大重叠 | 先加后减还是先减后加取决于端点开闭。 |

### 3.5 基础数学

| 板子 | 常见用法 | 注意点 |
|---|---|---|
| 快速幂 | 模幂、矩阵幂、倍增思想 | 乘法可能溢出时用 `__int128` 或快速乘。 |
| `gcd / lcm` | 约分、周期、数论题 | `lcm = a / gcd(a,b) * b`，先除后乘。 |
| 扩展欧几里得 | 求逆元、线性同余、丢番图方程 | 逆元存在条件是 `gcd(a,m)=1`。 |
| 费马逆元 | 组合数取模，`MOD` 为质数 | `a^(MOD-2) % MOD` 只适用于质数模且 `a` 不为 0。 |
| 线性筛/埃氏筛 | 质数表、质因数、欧拉函数 | `n` 不大用埃氏即可；多函数一起筛可用线性筛。 |
| 质因数分解 | 约数个数、最大质因子、分解统计 | 试除到 `sqrt(n)`；最后剩余大于 1 是质因子。 |
| 组合数预处理 | `C(n,k)` 多次查询 | 质数模下预处理 `fac` 和 `ifac`。 |

### 3.6 基础 DP

| 板子 | 常见用法 | 注意点 |
|---|---|---|
| 01 背包 | 每件物品最多选一次 | 容量倒序循环。 |
| 完全背包 | 每件物品可无限选 | 容量正序循环。 |
| 多重背包 | 每件有数量限制 | 二进制拆分或单调队列优化。 |
| LIS | 最长上升子序列、最少链覆盖变形 | `lower_bound` 对严格上升；`upper_bound` 对非降。 |
| 区间 DP | 合并石子、括号匹配、回文区间 | 按区间长度枚举，转移枚举断点。 |
| 树形 DP | 树上选择、树上最大独立集、换根 | 先 DFS 子树，再根据父子关系转移。 |
| 状压 DP | `n <= 20` 左右的集合状态 | 状态数 `2^n`；注意内存和转移枚举。 |
| 数位 DP | 统计 `[0,n]` 满足条件的数 | 记忆化参数通常含位置、限制、前导零、状态。 |

## 4. B 级：必须会套、会改

### 4.1 数据结构

| 板子 | 常见用法 | 注意点 |
|---|---|---|
| 并查集 DSU | 连通性、最小生成树、分组、朋友关系 | 路径压缩 + 按大小/秩合并；维护集合大小/权值可扩展。 |
| 带权并查集 | 相对关系、差值约束、食物链类 | 维护到父节点的权；合并时公式容易错，要画等式。 |
| 树状数组 Fenwick | 单点加区间和、逆序对、动态排名 | 下标从 1 开始；`lowbit`。 |
| 树状数组变形 | 区间加单点查、区间加区间查 | 两个 BIT 维护一次项和常数项。 |
| 线段树 | 区间查询、单点修改、区间最值/和 | 合并函数要和单位元匹配。 |
| 懒标记线段树 | 区间加/赋值、区间查询 | 多种懒标记共存时先后顺序最容易错。 |
| Sparse Table | 静态 RMQ、区间 gcd | 只能处理可重叠贡献的操作，如 min/max/gcd。 |
| 单调栈 | 最近更大/更小、矩形最大面积 | 出栈时计算贡献；严格/非严格比较要按题意。 |
| 单调队列 | 滑动窗口最值、DP 优化 | 队头维护合法范围，队尾维护单调性。 |
| Trie | 字符串前缀、01 异或最大值 | 节点数组大小估算：总字符数或 `n * 位数`。 |
| 分块/莫队 | 离线区间询问、修改少的统计 | 莫队适合增删一个端点可 O(1) 或 O(log n) 维护。 |

### 4.2 图论

| 板子 | 常见用法 | 注意点 |
|---|---|---|
| 链式前向星/邻接表 | 大图存边 | `vector<vector<pair<int,int>>>` 足够时优先用；超大图再用数组边。 |
| 拓扑排序 | DAG 判环、任务依赖、最长路 | 入度为 0 入队；结果数不足 `n` 说明有环。 |
| Dijkstra | 非负权最短路 | 不能处理负边；堆中状态可能过期。 |
| Bellman-Ford / SPFA | 负边、负环判定 | SPFA 可能被卡；负环判定用入队次数或松弛轮数。 |
| Floyd | `n <= 500` 左右全源最短路、传递闭包 | 三重循环顺序 `k,i,j`。 |
| Kruskal | 最小生成树 | 排边 + DSU；判断是否选了 `n-1` 条边。 |
| Prim | 稠密图 MST | 邻接矩阵版适合稠密图；堆优化版类似 Dijkstra。 |
| LCA 倍增 | 树上最近公共祖先、树上距离 | DFS/BFS 预处理深度和 `up[k][v]`。 |
| 树的直径 | 树上最长路 | 两次 BFS/DFS；带负权时要用树形 DP。 |
| Tarjan SCC | 有向图强连通缩点、2-SAT 基础 | `dfn/low/stack/in_stack`；缩点后是 DAG。 |
| 割点/桥 | 无向图关键点/边、边双/点双基础 | 根节点割点条件特殊；重边会影响桥判定。 |
| 二分图染色 | 判断二分图、奇环 | BFS/DFS 染两色；冲突说明非二分。 |
| 匈牙利算法 | 二分图最大匹配，中小规模 | 左侧逐个增广；复杂度通常 `O(nm)`。 |

### 4.3 字符串

| 板子 | 常见用法 | 注意点 |
|---|---|---|
| KMP 前缀函数 | 单模式串匹配、最小循环节 | `pi` 含义要统一；循环节条件 `n % (n - pi[n-1]) == 0`。 |
| Z 函数 | 模式匹配、前缀匹配长度 | 拼接 `pattern + '#' + text`。 |
| 字符串哈希 | 快速比较子串、回文判定、去重 | 双模或 `unsigned long long`；注意碰撞风险。 |
| Manacher | 最长回文子串、每个中心回文半径 | 统一奇偶长度；处理边界字符。 |
| AC 自动机 | 多模式串匹配、敏感词统计、DP 禁止串 | 建 fail 指针；统计时要沿 fail 链或拓扑累加。 |

### 4.4 数学进阶

| 板子 | 常见用法 | 注意点 |
|---|---|---|
| 中国剩余定理 CRT/exCRT | 多个同余方程合并 | 模数不互质时用 exCRT；检查可解条件。 |
| Lucas 定理 | 大 `n,k` 小质数模组合数 | 模数必须是质数。 |
| 矩阵快速幂 | 线性递推、路径计数 | 矩阵大小通常较小；乘法复杂度 `O(n^3 log k)`。 |
| 高斯消元 | 线性方程组、异或方程组、概率 DP | 浮点版注意 eps；异或版用 bitset 加速。 |
| 欧拉函数/莫比乌斯基础 | 互质计数、反演入门 | ICPC/CCPC 偶尔用，蓝桥杯较少。 |
| NTT/FFT | 多项式乘法、卷积计数 | NTT 需要合适质数；FFT 要处理精度。 |

## 5. C 级：高分冲刺和队伍模板建议放入

这些不一定要手写，但 ICPC/CCPC 队伍模板中建议有稳定版本。

| 板子 | 常见用法 | 注意点 |
|---|---|---|
| Dinic 最大流 | 二分图匹配变形、割、流量建模 | 建反向边；源汇、容量类型用 `long long` 时统一。 |
| 最小费用最大流 | 带费用匹配、运输、选择优化 | 费用可能为负；常用 SPFA 初始势能或 Johnson/Dijkstra 优化。 |
| 2-SAT | 布尔约束、二选一、矛盾判断 | 建蕴含图，SCC 后 `x` 与 `not x` 同 SCC 则无解。 |
| 重链剖分 HLD | 树上路径查询/修改 | 与线段树配合；边权转点权时处理 LCA 位置。 |
| 点分治 | 树上距离统计 | 处理经过当前重心的路径，递归子树；注意去重。 |
| 可持久化线段树 | 静态区间第 k 小、历史版本查询 | 离散化 + 主席树；节点数约 `n log n`。 |
| 可撤销并查集 | 离线动态连通性、分治加边 | 不能路径压缩，只按大小合并并记录修改栈。 |
| 后缀数组 SA | 后缀排序、最长公共前缀、不同子串数 | 模板长，适合放队伍文档。 |
| 后缀自动机 SAM | 不同子串、出现次数、最长公共子串 | 状态转移和 clone 细节多，建议用固定模板。 |
| 计算几何基础包 | 点、向量、叉积、凸包、线段相交 | eps 策略统一；整数坐标优先用 `long long` 叉积。 |
| 凸包 + 旋转卡壳 | 最远点对、凸多边形直径 | 先求凸包；点数小于 3 特判。 |
| 半平面交 | 线性约束可行区域 | 模板复杂、精度敏感；只建议队伍模板保留。 |
| 博弈 SG 函数 | 公平组合游戏、取石子变形 | 拆成独立子游戏异或；状态规模决定能否打表。 |

## 6. 按比赛整理自己的模板本

### 6.1 蓝桥杯 / 天梯赛优先版

建议先把下面内容背熟，能覆盖大部分基础与中档题：

1. C++ 快速 IO、常用类型、结构体排序。
2. 字符串处理：`substr/find/stoi/stoll`、字符分类、简单哈希。
3. 排序、二分、双指针、滑动窗口。
4. 前缀和、差分、二维前缀和、二维差分。
5. DFS、BFS、网格搜索、连通块。
6. 并查集。
7. 树状数组、基础线段树。
8. Dijkstra、Floyd、拓扑排序、Kruskal。
9. 01/完全/多重背包、LIS、区间 DP、树形 DP。
10. 快速幂、gcd、质数筛、组合数。

蓝桥杯特别注意：

- 日期题、进制题、模拟题、枚举题要练实现速度。
- 有些题可以暴力加剪枝或打表，不要过早套重算法。
- Java 选手要熟悉 `BigInteger`；C++ 选手要熟悉 `__int128`。

天梯赛特别注意：

- L1/L2 分数靠稳定实现，不要在简单题上浪费过多调试时间。
- L2 高频：排序模拟、map/set、BFS/DFS、并查集、最短路、树。
- L3 再考虑线段树、Tarjan、DP 优化、网络流。

### 6.2 ICPC / CCPC 队伍模板本建议目录

如果规则允许纸质 Team Reference Document，可以按下面顺序压缩：

1. 基础模板：IO、随机数、调试宏、常用 typedef、取模类。
2. 数据结构：DSU、Fenwick、Segtree、Lazy Segtree、Sparse Table、Trie、单调队列。
3. 图论基础：Dijkstra、Floyd、MST、Topo、LCA、树直径。
4. 图论进阶：Tarjan SCC/桥/割点、2-SAT、Dinic、MinCostFlow、二分图匹配。
5. 字符串：KMP、Z、Hash、Manacher、AC 自动机、SA/SAM。
6. 数学：快速幂、exgcd、CRT、组合数、Lucas、筛法、矩阵幂、高斯消元、NTT。
7. DP 优化：单调队列优化、斜率优化、分治 DP、状压常用枚举。
8. 计算几何：点线基础、凸包、旋转卡壳、圆、半平面交。
9. 杂项：莫队、可持久化线段树、可撤销并查集、随机化/哈希防卡。

队伍模板本不要只放代码，还要在每个板子前写：

- 适用条件。
- 时间复杂度。
- 输入下标是 0 还是 1。
- 需要开的数组大小。
- 常见坑。
- 一道自己通过过的测试题编号或链接。

## 7. 常见题面信号到算法映射

| 题面信号 | 优先考虑 |
|---|---|
| “最小的最大值”“最大的最小值”“是否存在一个 x” | 二分答案 + 判定。 |
| “区间多次查询，不修改” | 前缀和、Sparse Table、离线排序。 |
| “区间修改，区间查询” | 线段树懒标记、树状数组变形、差分离线。 |
| “动态加入边，问连通” | 并查集。 |

## 8. 背板子的验收标准

每个 A/B 级板子至少做到：

1. 能写出适用条件和复杂度。
2. 能解释核心不变量，例如 DSU 的父节点含义、线段树节点维护什么。
3. 能在 10 分钟内写出并通过一个最小样例。
4. 能列出 2 个常见坑。
5. 能手动构造边界样例：空、单点、全相等、极大值、负数、重边、自环、不可达。

建议练习方式：

- 每个板子找 2-3 道典型题，第一次看模板，第二次默写，第三次限时。
- 把 WA 的坑写在模板注释里，不要只改题解代码。
- 每次赛前只复习自己的稳定版本，不临场换新模板。

## 9. 不建议优先背的内容

这些算法不是没用，而是不适合作为入门阶段背诵重点：

- Link-Cut Tree、动态树：出现少，模板复杂，维护成本高。
- 后缀树：多数竞赛场景可用 SA/SAM/哈希替代。
- 高阶多项式全家桶：除非目标是强校队/区域赛金牌以上。
- 复杂计算几何全集：先保证叉积、凸包、线段相交、点到线距离稳定。
- 过度宏化的个人模板：省不了多少时间，反而增加调试难度。

## 10. 最小背诵清单

如果时间有限，按这个顺序背：

1. 快速 IO、类型、排序、二分、双指针。
2. 前缀和、差分、离散化。
3. DFS、BFS、01 BFS。
4. 并查集。
5. 树状数组。
6. 线段树基础版。
7. Dijkstra、Floyd、Kruskal、拓扑排序。
8. LCA。
9. KMP、字符串哈希。
10. 01/完全/多重背包、LIS、区间 DP、树形 DP、状压 DP。
11. 快速幂、exgcd、筛法、组合数。
12. Tarjan SCC、桥/割点。
13. Dinic 最大流。
14. Manacher、AC 自动机。
15. 计算几何基础包、凸包。

蓝桥杯/天梯赛做到前 11 项已经很有竞争力；ICPC/CCPC 建议至少做到前 15 项，并把 C 级中自己队伍薄弱的内容补进模板本。
